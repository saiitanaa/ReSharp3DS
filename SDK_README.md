## Latest SDK Update

The SDK documentation now reflects the current runtime API and launcher manifest system.

Highlights:

- Manifest-based app metadata.
- Folder-based `.pe` apps.
- New helper types: `Button`, `ScreenTarget`, `Vector2`, `Rect`, `Timer`, `GameApp`.

## ReSharp3DS SDK is the managed C# SDK used to create applications for the ReSharp3DS runtime.

This branch is dedicated to writing C# applications.
The native Nintendo 3DS homebrew runtime is maintained separately in the main homebrew branch.

With this SDK, you create a C# nanoFramework project, build it into an `app.pe` file, then copy that file to the SD card so it can be loaded by the ReSharp3DS runtime.

## What this SDK is for

This SDK is used to create C# applications for ReSharp3DS.

A ReSharp3DS application is not a classic Windows or desktop C# app.
It is a nanoFramework C# application compiled into a `.pe` file and loaded by the Nintendo 3DS runtime.

The final application file must be named:

```txt
app.pe
```

The runtime expects it here:

```txt
sdmc:/ReSharp3DS/app.pe
```

The runtime also needs the matching nanoFramework core library:

```txt
sdmc:/ReSharp3DS/mscorlib.pe
```

Current runtime versions can also browse folders inside:

```txt
sdmc:/ReSharp3DS/
```

This means `.pe` applications can also be placed in subfolders, for example:

```txt
sdmc:/ReSharp3DS/MyApp/MyApp.pe
```

Current runtime versions use this path for the nanoFramework core library:

```txt
sdmc:/ReSharp3DS/bin/mscorlib.pe
```

## Branch structure

The ReSharp3DS project is split into multiple branches.

```txt
main / homebrew branch
```

Contains the native Nintendo 3DS homebrew runtime.
This is the part launched from the Homebrew Launcher on the 3DS.

```txt
sdk branch
```

Contains the C# SDK files, examples and project setup used to create `app.pe`.

Use this branch if you want to create a C# program for ReSharp3DS.

## Requirements

To create a ReSharp3DS program, you need:

* Windows
* Visual Studio
* .NET nanoFramework Extension for Visual Studio
* A nanoFramework C# project
* The ReSharp3DS SDK files
* The ReSharp3DS runtime from the homebrew branch
* A Nintendo 3DS with homebrew access, or Citra for testing

## Installing the nanoFramework extension

Open Visual Studio.

Go to:

```txt
Extensions > Manage Extensions
```

Search for:

```txt
nanoFramework
```

Install:

```txt
.NET nanoFramework Extension
```

Restart Visual Studio after the installation.

After restarting Visual Studio, nanoFramework project templates and build support should be available.

## Creating a new ReSharp3DS program

Create a new nanoFramework C# application in Visual Studio.

Your project should produce an executable nanoFramework application.
The output file generated by the build will be used as the ReSharp3DS application.

The expected output is:

```txt
app.pe
```

This file must be copied to:

```txt
sdmc:/ReSharp3DS/app.pe
```

Current runtime versions can also launch `.pe` files from folders inside:

```txt
sdmc:/ReSharp3DS/
```

Example:

```txt
sdmc:/ReSharp3DS/MyApp/MyApp.pe
```

## nanoFramework version

ReSharp3DS depends on the nanoFramework version used by the runtime.

Use the nanoFramework version required by the ReSharp3DS release you are targeting.

Recommended version for the current SDK setup:

```txt
nanoFramework.CoreLibrary 1.9.0
```

If the release specifically provides or requires:

```txt
nanoFramework.CoreLibrary 1.9.0-preview.11
```

then use that exact version instead.

The version used by your C# project must match the `mscorlib.pe` used by the ReSharp3DS runtime.

Do not update the nanoFramework package randomly unless the runtime and `mscorlib.pe` are updated too.

A version mismatch can cause:

```txt
ResolveAll errors
PrepareForExecution errors
runtime crashes
missing methods
native calls not working
heap errors
```

## Project setup

Your project should include the SDK API file:

```txt
ReSharp3DS.cs
```

This file gives your program access to the ReSharp3DS API.

Your project should also include your application file:

```txt
Program.cs
```

Recommended project file order:

```txt
ReSharp3DS.cs
Program.cs
Properties/AssemblyInfo.cs
```

Important: do not compile two copies of `ReSharp3DS.cs` in the same project.
If your app has both `ReSharp3DS.cs` and `app/ReSharp3DS.cs`, remove one of them.

```txt
  Properties/
```

```json
  "author": "Saysaa",

  "entry": "app.pe",
  "description": "Application de test pour ReSharp3DS"
```

After building, copy the generated `.pe` file and manifest to your SD card.

## SD card layout

Recommended current layout:

```txt
sdmc:/3ds/ReSharp3DS.3dsx
sdmc:/ReSharp3DS/bin/mscorlib.pe
sdmc:/ReSharp3DS/apps/MyApp/manifest.json
sdmc:/ReSharp3DS/apps/MyApp/app.pe
sdmc:/ReSharp3DS/apps/MyApp/assets/sprite.bmp
sdmc:/ReSharp3DS/apps/MyApp/assets/sound.wav
```

The launcher scans:

```txt
sdmc:/ReSharp3DS/
```

and shows folders and `.pe` applications.

If `manifest.json` contains:

```json
{
  "name": "My App",
  "author": "Saysaa",
  "version": "0.1.0",
  "entry": "app.pe"
}
```

then the launcher can display `app.pe` as `My App` and show the author/version metadata when selected.

## App Manifest

Current launcher builds support an optional `manifest.json` next to a `.pe` file.

Example:

```json
{
  "name": "App de test",
  "author": "Saysaa",
  "version": "0.1.0",
  "entry": "app.pe",
  "description": "Application de test pour ReSharp3DS"
}
```

Expected SD structure:

```txt
sdmc:/ReSharp3DS/apps/TestApp/
  manifest.json
  app.pe
```

The launcher reads the `entry` field and applies the manifest metadata to that `.pe` file.
For example, `app.pe` can be displayed as `App de test`.

When the app is selected, the launcher can also display:

```txt
Author: Saysaa  Version: 0.1.0
```


## Writing your first program

Create or edit `Program.cs`.

Example:

```csharp
namespace ReSharp3DS
{
    public class Program
    {
        static bool initialized = false;

        public static void Main()
        {
            if (!initialized)
            {
                initialized = true;

                Console.Clear();
                Console.WriteLine("Hello from ReSharp3DS!");
                Console.WriteLine("Press A or START.");
            }

            if (Input.IsAPressed())
            {
                Console.WriteLine("A pressed");
            }

            if (Input.IsStartPressed())
            {
                Console.WriteLine("START pressed");
            }

            Runtime.Yield();
        }
    }
}
```

## Important: how `Main()` works

In ReSharp3DS, `Program.Main()` is currently used like a repeated update function.

That means it can be called many times while the application is running.

Use `static` fields to keep your program state.

Recommended pattern:

```csharp
namespace ReSharp3DS
{
    public class Program
    {
        static bool initialized = false;
        static int frame = 0;

        public static void Main()
        {
            if (!initialized)
            {
                initialized = true;

                Console.Clear();
                Console.WriteLine("App started");
            }

            frame++;

            Runtime.Yield();
        }
    }
}
```

Avoid clearing the screen every time `Main()` is called.

Bad pattern:

```csharp
public static void Main()
{
    Console.Clear();
    Console.WriteLine("This can flicker.");

    Runtime.Yield();
}
```

Good pattern:

```csharp
static bool initialized = false;

public static void Main()
{
    if (!initialized)
    {
        initialized = true;

        Console.Clear();
        Console.WriteLine("Draw once.");
    }

    Runtime.Yield();
}
```

## Time API

The Time API can be used for timers, cooldowns and animations.

```csharp
int ms = Time.Milliseconds();
int seconds = Time.Seconds();
```

Example:

```csharp
static int lastShot = 0;

public static void Main()
{
    int now = Time.Milliseconds();

    if (Input.IsAPressed() && now - lastShot > 250)
    {
        Console.WriteLine("Shoot!");
        lastShot = now;
    }

    Runtime.Yield();
}
```

Example:

```csharp

{
    Graphics.FillRect(Touch.X(), Touch.Y(), 8, 8, 0xFFFFFF);
```

## Building your program

In Visual Studio:

1. Restore NuGet packages.
2. Clean the solution.
3. Rebuild the solution.
4. Find the generated `app.pe`.
5. Copy it to:

```txt
sdmc:/ReSharp3DS/app.pe
```

Then launch the ReSharp3DS runtime on your Nintendo 3DS.

If you are using the launcher file browser, you can also copy your `.pe` file to a subfolder:

```txt
sdmc:/ReSharp3DS/MyApp/MyApp.pe
```

## Running your program on 3DS

1. Build your C# project.
2. Copy `app.pe` to:

```txt
sdmc:/ReSharp3DS/app.pe
```

3. Make sure `mscorlib.pe` exists here:

```txt
sdmc:/ReSharp3DS/mscorlib.pe
```

Current runtime versions use:

```txt
sdmc:/ReSharp3DS/bin/mscorlib.pe
```

4. Start the Homebrew Launcher.
5. Launch ReSharp3DS.
6. Start the runtime from the ReSharp3DS menu.

Your C# program should now run on the console.

## Updating your program

Every time you change your C# code:

1. Rebuild the nanoFramework project.
2. Copy the new `app.pe` to the SD card.
3. Replace the old `sdmc:/ReSharp3DS/app.pe`.
4. Relaunch the runtime.

If the old behavior still appears, you probably copied the wrong `app.pe` or forgot to rebuild the project.

If you are using a custom `.pe` file inside a subfolder, replace that file instead.

## Troubleshooting

### The app stays on `[CLR] Execute`

Check that:

```txt
app.pe exists in sdmc:/ReSharp3DS/app.pe
mscorlib.pe exists in sdmc:/ReSharp3DS/mscorlib.pe
the C# project was rebuilt
the correct app.pe was copied
the nanoFramework version matches the runtime
```

For current runtime versions, also check:

```txt
mscorlib.pe exists in sdmc:/ReSharp3DS/bin/mscorlib.pe
your .pe file exists inside sdmc:/ReSharp3DS/
```

### The app does not update after changes

You probably launched an old `app.pe`.

Rebuild the project and copy the new file again:

```txt
app.pe -> sdmc:/ReSharp3DS/app.pe
```

If you are using a subfolder, replace the `.pe` file inside that subfolder.

### Text flickers

Do not call `Console.Clear()` every tick.

Use a `static bool initialized` field and draw the initial text only once.

### Inputs do not work

Check that:

```txt
you are using the correct ReSharp3DS.cs file
your runtime version supports the input methods
you rebuilt and copied the latest app.pe
```

### BMP does not draw

Check that the file is:

```txt
BMP
24-bit or 32-bit
uncompressed
```

Also check that the path is relative to the running `.pe` file.

### Audio does not work on real hardware

Make sure DSP has been dumped correctly and is available on the SD card.

### Build fails in Visual Studio

Check that:

```txt
the .NET nanoFramework Extension is installed
Visual Studio was restarted after installing the extension
NuGet packages were restored
the project uses the correct nanoFramework.CoreLibrary version
```

### Runtime errors after updating nanoFramework packages

The SDK project and the runtime must use compatible nanoFramework files.

If you update the NuGet package, you may also need to update:

```txt
mscorlib.pe
the runtime native side
the SDK API bindings
```

For now, use the nanoFramework version recommended by the ReSharp3DS release.
